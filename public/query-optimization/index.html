Query path

Query optimization starts by finding the SQL that affects real work.

Query optimization is a broad database optimization task: identify which checkout, booking, inventory, or report query is putting pressure on the database before changing indexes or infrastructure.

Total time

High total time points to repeated business impact, even when each query feels only moderately slow.

Mean time

High mean time points to heavy individual queries that may block important workflows.

Temp writes

Temp files can signal sort or join pressure that deserves EXPLAIN review.

The broad SQL performance tuning guide helps rank statements before engine-specific review, and index optimization keeps index changes tied to evidence.
